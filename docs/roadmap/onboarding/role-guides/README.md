# Role Guides

这组文档把 onboarding 再往前推进一层，不再只回答“怎么进入项目”，而是回答“按我的职责，应该先看什么、先做什么、先避开什么”。

## 适用方式

如果你已经完成了 onboarding 的前两步：

- [../01-first-30-minutes.md](../01-first-30-minutes.md)
- [../02-first-task-playbook.md](../02-first-task-playbook.md)

接下来就可以按自己的职责进入对应 guide。

## 角色入口

- [01-foundation-engineer.md](./01-foundation-engineer.md)
  适合主题、token、图标、i18n、overlay、shared primitives 方向。
- [02-form-engineer.md](./02-form-engineer.md)
  适合输入控件、筛选控件、字段状态统一、表单集成方向。
- [03-shell-engineer.md](./03-shell-engineer.md)
  适合壳层、导航、多列布局、对象页框架方向。
- [04-chart-engineer.md](./04-chart-engineer.md)
  适合 microchart、KPI card、图形语义和可视化交互方向。
- [05-docs-engineer.md](./05-docs-engineer.md)
  适合 docs 演示站、lab 结构、walkthrough 场景和集成示例方向。
- [06-qa-accessibility-engineer.md](./06-qa-accessibility-engineer.md)
  适合 smoke tests、keyboard path、a11y、回归验证和验收流程方向。

## 这些 guide 会帮你回答什么

- 这个角色的主要目标和边界是什么
- 最常改哪些目录和文件
- 第一批更适合承接哪些任务
- 最容易踩的坑是什么
- 做完一个任务至少应该怎么验证

## 建议阅读顺序

1. 先看你自己的角色 guide
2. 再看相邻协作角色的 guide
3. 最后回到 [../../backlog/owner-backlog-board.md](../../backlog/owner-backlog-board.md) 和 [../../execution/weekly-execution-board.md](../../execution/weekly-execution-board.md) 对齐当前节奏
