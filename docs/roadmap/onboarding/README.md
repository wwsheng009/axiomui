# AxiomUI Onboarding

这组文档给第一次进入 AxiomUI 仓库的成员使用，目标是在 `30-60 分钟` 内建立基本上下文，并顺利接到第一个任务。

如果你已经熟悉这套 roadmap 文档，也可以把这里当成一个压缩入口，从这里跳回 backlog、执行板和 meeting kit。

## 建议阅读顺序

1. [01-first-30-minutes.md](./01-first-30-minutes.md)
   用最短路径把环境、仓库结构、当前执行节奏和 docs 演示站先跑起来。
2. [02-first-task-playbook.md](./02-first-task-playbook.md)
   把“选任务 -> 建 issue -> 开发 -> 验收 -> 提 PR”串成一条可操作路径。
3. [03-doc-map.md](./03-doc-map.md)
   解决“我应该看哪份文档”的问题，按场景回跳 roadmap、backlog、execution 和 meetings。
4. [role-guides/README.md](./role-guides/README.md)
   按职责进入 `Foundation / Form / Shell / Chart / Docs / QA` 的角色上手包。
5. [checklists/README.md](./checklists/README.md)
   按 `Day 1 / Week 1 / First PR / First Review` 使用执行清单。
6. [04-demo-walkthrough-checklist.md](./04-demo-walkthrough-checklist.md)
   作为本地验收和 walkthrough 的最小检查表。

## 这组文档适合谁

- 新加入项目的开发成员
- 刚接手某个 sprint 的协作者
- 需要快速进入上下文的 reviewer
- 需要主持周会或 milestone 验收的 owner

## Onboarding 完成标准

完成这组 onboarding 后，成员应该能回答下面这些问题：

- 当前项目的主目标、阶段顺序和最近里程碑是什么
- 现在应该从哪个 sprint 或哪个 owner 视图选任务
- 一个任务的最小交付物包括哪些内容
- docs 演示站应该从哪些页面验证主题、表单、壳层和图形
- issue、PR、周会和 milestone 验收分别该用哪份模板
