# Onboarding Checklists

这组清单把 onboarding 文档再往执行层推进一步，适合在真正开始协作时逐条勾掉。

如果说：

- [../01-first-30-minutes.md](../01-first-30-minutes.md) 解决的是“怎么快速进入上下文”
- [../02-first-task-playbook.md](../02-first-task-playbook.md) 解决的是“怎么接第一个任务”
- [../role-guides/README.md](../role-guides/README.md) 解决的是“按角色该怎么工作”

那么这里解决的是“我今天、这周、第一次提 PR、第一次做 review 时到底该检查什么”。

## 清单入口

- [01-day-1-checklist.md](./01-day-1-checklist.md)
  第一天上手清单，适合刚进入项目时逐条确认。
- [02-week-1-checklist.md](./02-week-1-checklist.md)
  第一周协作清单，适合从观察期过渡到稳定承接任务。
- [03-first-pr-checklist.md](./03-first-pr-checklist.md)
  第一次提 PR 清单，帮助你在提交流程前把 docs、验证和依赖收口。
- [04-first-review-checklist.md](./04-first-review-checklist.md)
  第一次做 review 清单，帮助你按 AxiomUI 的质量标准做反馈。

## 建议使用方式

1. 第一天先过 [01-day-1-checklist.md](./01-day-1-checklist.md)
2. 开始选任务后切到 [02-week-1-checklist.md](./02-week-1-checklist.md)
3. 第一次提 PR 前过 [03-first-pr-checklist.md](./03-first-pr-checklist.md)
4. 第一次帮别人 review 前过 [04-first-review-checklist.md](./04-first-review-checklist.md)
