# Week 1 Checklist

这份清单适合新成员进入协作的第一周，用来把“旁观”推进到“稳定承接任务”。

## 任务承接

- [ ] 已确认一个主任务和一个备选任务
- [ ] 已确认主任务的主依赖是否完成
- [ ] 已确认该任务应该落在哪个 sprint 或 owner 视图里
- [ ] 已知道这个任务会改哪些目录和文件

## 协作对齐

- [ ] 已阅读对应角色 guide
- [ ] 已和相邻角色对齐边界，例如 `Foundation / Form` 或 `Shell / Docs`
- [ ] 已确定 docs 写入位置，不会长期堆在单个大文件里
- [ ] 已知道这个任务如果出现返工，应该回写到哪份文档

## 实现准备

- [ ] 已查看 [../../02-api-conventions.md](../../02-api-conventions.md)
- [ ] 已查看 [../../03-risk-register.md](../../03-risk-register.md)
- [ ] 已确认是否存在共享 primitive 可以复用
- [ ] 已知道需要补哪些 docs 示例和 walkthrough 场景

## 第一周建议产出

- [ ] 至少完成一个低到中等复杂度任务
- [ ] 至少提交一个包含 docs 变化的 PR
- [ ] 至少参加一次 weekly sync 或 walkthrough
- [ ] 至少更新一次 backlog、execution 或 decision/risk 文档

## Week 1 完成标准

第一周结束时，理想状态是：

- 你已经不再只依赖口头说明来找任务
- 你知道如何自己定位依赖、验收点和 demo 入口
- 你已经完成一次完整的 issue 到 PR 闭环
