# Execution Docs

这组文档用于把 backlog 和 owner 视图继续收口成周执行计划，方便团队从 roadmap 直接进入每周交付。

截至 `2026-03-27`，这组文档分成两类用途：

- `current-status.md`
  反映当前代码快照的真实实施状态，适合做 live status 判断
- `weekly-execution-board.md`
  保留 `Sprint 1-5` 的周计划拆分方式，适合作为历史节奏参考，不再单独承担 `Sprint 6` 的实时状态同步

## 文档目录

- [current-status.md](./current-status.md)
  基于当前代码快照的实施情况汇总，适合在排班前先确认真实项目进度。
- [weekly-execution-board.md](./weekly-execution-board.md)
  按周拆分的执行看板，当前覆盖 `Sprint 1-5` 的历史周计划基线：`Week 1-4` 的详细周计划，以及 `Week 5-10` 的建议节奏。

## 建议使用方式

1. 周会前先看 [current-status.md](./current-status.md) 确认真实代码进度。
2. 再看 [master-backlog-board.md](../backlog/master-backlog-board.md) 和 [owner-backlog-board.md](../backlog/owner-backlog-board.md) 确认当前 backlog 和 owner 分布。
3. 若需要参考旧周节奏，再回看 [weekly-execution-board.md](./weekly-execution-board.md)。
4. 按 `Owner` 和 `Priority` 分派本周任务。
5. 每周结束回写完成情况、阻塞项和下一周调整。
6. 若任务状态变化较大，优先回到 [master-backlog-board.md](../backlog/master-backlog-board.md) 与 [current-status.md](./current-status.md) 同步全局状态。
