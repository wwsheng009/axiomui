# Execution Docs

这组文档用于把 backlog 和 owner 视图继续收口成周执行计划，方便团队从 roadmap 直接进入每周交付。

## 文档目录

- [current-status.md](./current-status.md)
  基于当前代码快照的实施情况汇总，适合在排班前先确认真实项目进度。
- [weekly-execution-board.md](./weekly-execution-board.md)
  按周拆分的执行看板，当前覆盖 `Week 1-4` 的详细周计划，并给出后续周的建议节奏。

## 建议使用方式

1. 周会前先看 [current-status.md](./current-status.md) 确认真实代码进度。
2. 再看 [weekly-execution-board.md](./weekly-execution-board.md) 确认本周目标。
3. 按 `Owner` 和 `Priority` 分派本周任务。
4. 每周结束回写完成情况、阻塞项和下一周调整。
5. 若任务状态变化较大，回到 [master-backlog-board.md](../backlog/master-backlog-board.md) 同步全局状态。
