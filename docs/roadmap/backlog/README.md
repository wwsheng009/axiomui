# Backlog Drafts

这组文档用于把 roadmap 和 sprint checklist 继续细化成可直接复制为 issue 的 backlog 草稿。

## Sprint Backlogs

- [master-backlog-board.md](./master-backlog-board.md)
  六个 sprint 的总控板，汇总 `epic / owner / status / dependency / draft`。
- [owner-backlog-board.md](./owner-backlog-board.md)
  按 owner 重组的总览，适合直接做团队分工。
- [completion-snapshot.md](./completion-snapshot.md)
  截至 `2026-03-27` 的 `Sprint 1-5` 完成快照，按 sprint 和 owner 汇总交付结果与完成度。
- [sprint-1/README.md](./sprint-1/README.md)
  Foundation sprint 的 epic 与任务草稿。
- [sprint-2/README.md](./sprint-2/README.md)
  Form Core sprint 的 epic 与任务草稿。
- [sprint-3/README.md](./sprint-3/README.md)
  Advanced Filters sprint 的 epic 与任务草稿。
- [sprint-4/README.md](./sprint-4/README.md)
  Shell And Object Page sprint 的 epic 与任务草稿。
- [sprint-5/README.md](./sprint-5/README.md)
  MicroChart sprint 的 epic 与任务草稿。
- [sprint-6/README.md](./sprint-6/README.md)
  Workflow And Planning sprint 的 backlog 与实施台账，当前已完成 `S6-01`、`S6-02`、`S6-08`。

## 使用方式

1. 从对应 sprint 的 `README.md` 进入任务列表。
2. 复制任务草稿到 GitHub issue 或项目管理工具。
3. 用 `.github/ISSUE_TEMPLATE/` 下的模板做最终收口。
4. 在实施过程中回写 docs、测试和验收备注。
